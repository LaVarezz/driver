Общая документация.
------------------------------------------- Space ----------------------------------------------------

- BATTLE - 
Battle - Вся логика боя.
battlequeue - Очередь боя.
core - Фазовая логика боя.
updates - Обновление и отображение обьектов в боевом режиме.
Почти весь процесс обработки боя происходит в core, создается в battle_scene(см. engine). 

- ENGINE - 
Controllers содержит папку binds и файл events. 
binds - Различные наборы управлений (Боевой режим и обычный)
events - обработка нажатий кнопок, использует event_handler. Работает на протяжении всего игрового цикла и во всех сценах.

Debug - содержит папку logs, файлы debug state, debug actions.
logs отвечает за логирование действий игры с момента запуска.
debug state - состояния дебаг-меню
debug_actions действия в режиме отладки - пока пусто.

managers - содержит папку managers и файл manager.
В папке хранятся локальные менеджеры(Виджетов, камеры и тд).
manager содержит все локальные менеджеры, все действия с ними строго через него(не трожь!)

scenes - содержит две папки, scenes и scene_loops.
scenes - Содержит классы сцен с их методами настройки и запуска цикла(scene_loop)
scene_loops - циклы сцен, обрабатываются в scenes.

Отвечает за логику самой игры и набор сцен.

GAME_OBJECTS
Содержит папку ships и cells.
каждая из них содержит базовый класс корабля и клетки и его подклассы в папках. Создаются в generation(см. scripts)

- RESOURSES - 
три папки: backgrounds, sprites, cache.
первые две хранят изображения в формате png для фона и игровых обьектов. cache содержит папку cached_text. cached_text при создании рендерит весь алфавит 1 раз и больше не тратит ресурсы на рендер. при создании можно обратиться с методом создания текста, вернет обьект, которому нужно вызвать метод draw(). Значительно экономит ресурсы на отрисовке.

- SCRIPTS -
4 папки. calculators, generation, loaders, utils.
первая папка содержит скрипты, рассчитывающие какие-то параметры, вторая принимает на вход параметры и создает какой-либо обьект из папки game_objects. третья папка оперирует загрузками ресурсов, четвертая - быстрые и не большие изменения в программе. (например, сменить направление корабля)

- SETTINGS -
4 папки. configs, constants, protocols, settings.  первая содержит конфигурации для создания сцены загрузки. вторая содержит все константы игры. третья - шаблоны и утиные протоколы для внутреннего использования. четвертая папка содержит папки с настройками каждой сцены. в каждой: env(быстрая и внешняя настройка), @_settings - присванивание рассчитанных настроек, calculator для рассчета параметров и config для присваивания жестко заданных параметров.

- UI -
3 папки. debug содержит всего один файл - debug_render. отображает dev-интерфейс. Wudgets хранит базовый виджет и типы виджетов(для кнопки отдлельная папка с классом кнопки и командами к ней). Animation отвечает за визуальное отображение эффектов - с этим классорм работает менеджер анимаций. а вот конфигурации animation(например, анимация движения) содежится в папке animation_effects.

В целом отвечает за визуальное отображение программы. 
------------------------------------------- Space ----------------------------------------------------
Цепочка наследования: Главный класс main. Оттуда запускается игра и он содержит manager, управляющего локальными менеджерами. содержит ссылку на сцены и переключание сцен проходит в main, как и выход из него. И в целом mainb имеет поллный доступ ко всем аспектам игры. Это движок.
От него цепочка управления переходит к scene, которая запускает цикл сцены(в т.ч цикл загрузки).
main-scene-scene_loop
main-manager.animation_handler ect.
main.settings.window_settings ect.
